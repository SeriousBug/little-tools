## Package manager

Use **bun**.

## Commands

- `bun run dev` — Vite dev server (long-running)
- `bun run build` — `tsc -b && vite build`; also generates per-route HTML and OG PNGs
- `bun run lint` / `bun run format`
- `bun run test` — Vitest single run (watch is off in `vite.config.ts`); `bun run test:watch` for watch
- Single test: `bunx vitest run src/base64/base64.test.tsx` or `-t "<name>"`

`bun run prepare` runs `panda codegen`; rerun it manually if `@styled-system/*` imports break after editing `panda.config.ts`.

## Architecture

Client-only React 19 + TypeScript SPA. Every tool runs entirely in the browser. Deployed to Cloudflare Pages.

### Tool registry (`src/tools.ts`)

`TOOLS` is the single source of truth for each tool: route path, slug, sidebar label/icon, `<title>`, meta description, OG card text. To add a tool:

1. Add an entry to `TOOLS`.
2. Add the icon name to `IconName` in `src/components/icon-names.ts` and import the SVG in `src/components/Icon.tsx`.
3. Create `src/routes/<slug>.tsx` (file-based route). `src/routeTree.gen.ts` is regenerated by the plugin — never edit it.
4. Put tool logic under `src/<slug>/` (see `base64/` as the smallest reference).

The sidebar (`src/routes/__root.tsx`) and the SEO build plugin both iterate `TOOLS`.

### Routing

TanStack Router with the Vite plugin (`autoCodeSplitting: true`). `__root.tsx` defines the persistent sidebar/footer layout.

### Styling — Panda CSS

`import { css } from '@styled-system/css'` with style objects. Theme tokens are in `panda.config.ts`; semantic tokens (`bg.canvas`, `fg`, `accent`, …) auto-resolve dark mode via `_dark`. Use semantic tokens so components don't branch per theme.

Dark mode toggles a `light`/`dark` class on `<html>` (see `src/theme.ts` and the inline boot script in `index.html` that prevents FOUC).

### State

Zustand. Per-tool stores live next to the tool; global UI state in `src/theme.ts` and `src/sidebar.ts`.

### Icons — custom Vite plugin

`vite-plugin-svg-icon.ts` handles `import foo from './foo.svg?icon'`: rewrites black fills/strokes to `currentColor` and exports the SVG markup as a string. `Icon.tsx` injects it via `dangerouslySetInnerHTML` so it inherits text color.

### SEO plugin

`vite-plugin-seo.ts` runs in `closeBundle`. For each `TOOLS` entry it renders an OG PNG (`dist/og/<slug>.png`) and writes `dist/<slug>/index.html` with tool-specific `<title>` and meta tags, replacing the block between `<!-- SEO:META -->` and `<!-- /SEO:META -->`. **Don't remove those markers in `index.html`** — the build throws without them.

### Testing

Vitest + happy-dom + `@testing-library/react`. Tests colocated as `*.test.tsx`.
